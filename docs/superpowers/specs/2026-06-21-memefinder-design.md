# MemeFinder — 設計文件（Design Spec）

- 日期：2026-06-21
- 狀態：已與使用者確認，待寫實作計畫
- 靈感來源：[ShiQu1218/MemeTalk](https://github.com/ShiQu1218/MemeTalk)（Python/Streamlit 的本地迷因語意搜尋系統；**注意：MemeTalk 不內建線上圖庫，使用者需自備迷因資料夾**）

## 1. 目標（What & Why）

做一個**原生 SwiftUI macOS App**：使用者指定一個本機「迷因圖資料夾」，App 用 Gemini 視覺模型自動幫每張圖建立索引（讀出圖中文字、生成中文描述與情緒/主題標籤），之後使用者在搜尋框輸入中文 → 出現最相關的迷因圖 → **點一下即複製到剪貼簿**，可直接貼到 LINE / Slack / Messages 等。

成功標準：
- 指定資料夾後能自動完成索引並顯示進度。
- 輸入中文關鍵字/語意敘述能回傳相關度排序的迷因圖。
- 點擊任一結果，圖片即進入系統剪貼簿，可在其他 App 貼上。
- 所有列出的功能皆有對應自動化測試且通過。

## 2. 範圍（Scope）

### 第一版要做（V1）
- 本機資料夾來源（hybrid 架構的本機路徑）。
- Gemini 視覺自動標註 + Gemini 語意嵌入。
- 語意向量 + 關鍵字混合搜尋。
- 結果縮圖牆 + 點擊複製到剪貼簿。
- 設定頁：選資料夾、輸入/管理 Gemini API 金鑰。

### 第一版不做（YAGNI，架構預留接口）
- 線上來源（Tenor / Giphy 等第三方圖庫）。
- 全域快捷鍵、選單列常駐。
- 以圖搜圖。

## 3. 架構與元件

各元件單一職責、透過明確介面溝通、可獨立測試。Gemini 相關呼叫一律藏在 protocol 後方以便注入 mock。

1. **Settings 設定**
   - 選擇迷因資料夾：使用 security-scoped bookmark 持久化授權。
   - Gemini API 金鑰：輸入欄位放在**設定頁**，介面以遮蔽方式顯示；實際金鑰存入 macOS Keychain（不以明文落地）。
   - 介面：提供「重新索引」按鈕與索引狀態顯示。

2. **Indexer 索引器**
   - 掃描資料夾中的影像（`.jpg / .jpeg / .png / .webp`）。
   - 對每張**尚未索引**的圖（以「檔案路徑 + 修改時間」判斷）：
     - 呼叫 Gemini 視覺模型 → 回傳 JSON：`{ ocr_text, description, tags[], emotion }`。
     - 呼叫 Gemini 嵌入模型 → 取得語意向量（float 陣列）。
   - 將結果寫入本機索引檔。已索引者跳過。
   - 對外回報進度（已處理/總數）與錯誤清單。

3. **Search 搜尋**
   - 輸入做 debounce。
   - 將查詢字串經 Gemini 嵌入成向量。
   - 對索引中所有向量計算餘弦相似度，並對 `ocr_text / tags` 命中關鍵字者加權，合併排序。
   - 回傳排序後的結果（含分數，供測試驗證）。

4. **Results Grid 結果牆**
   - 以縮圖網格呈現排序結果。
   - 縮圖延遲載入，避免一次載入全部原圖。

5. **Clipboard 複製器**
   - 點擊結果 → 讀取原圖 → 以影像格式寫入 `NSPasteboard`。
   - 顯示短暫「已複製 ✓」提示。

## 4. 資料流

```
資料夾 → Indexer(Gemini 視覺 + 嵌入) → 本機索引檔(JSON @ App Support)
查詢字串 → Gemini 嵌入 → 餘弦相似度排序(+ 關鍵字加權) → 結果牆 → 點圖 → 系統剪貼簿
```

## 5. 搜尋策略（決策）

採 **語意向量 + 關鍵字混合**：
- 純關鍵字成本低但中文語意召回差。
- 語意向量能支援「打相關敘述就找到圖」的需求；嵌入呼叫成本極低。
- 最終分數 = 餘弦相似度 + 關鍵字命中加權（權重於實作時調校，並以測試固定行為）。
- 替代方案（純關鍵字）已評估並否決，理由為中文語意召回不足。

## 6. 儲存

| 資料 | 位置 |
|------|------|
| Gemini API 金鑰 | macOS Keychain |
| 資料夾授權 | security-scoped bookmark（存 UserDefaults） |
| 索引（標籤、文字、向量） | App Support 下單一 JSON 檔 |

索引採 JSON：個人收藏量級足夠，依賴最少；日後規模變大再遷移至 SQLite（介面預留）。

## 7. 錯誤處理

- API 金鑰缺失/無效 → 導向設定頁並提示。
- 索引時網路失敗 → 標記該圖為待重試，不中斷整批。
- 圖片讀取失敗/格式不支援 → 跳過並記錄於錯誤清單。
- 查無結果 → 顯示友善提示（建議重新索引或換關鍵字）。

## 8. 測試（每個功能皆須對應測試並通過）

- 索引 JSON 解析（正常 / 缺欄位 / 格式錯誤）。
- 餘弦相似度計算與排序正確性。
- 關鍵字加權合併排序行為。
- 索引檔存取（存檔 / 載入 / 損毀回復）。
- 增量索引：已索引項目正確跳過。
- 剪貼簿寫入（透過可注入介面驗證）。
- Gemini 呼叫以 protocol + mock 隔離，不打真實 API。

## 9. 建置方式

- 以 Swift Package Manager 建置 SwiftUI 執行檔。
- 以腳本包成 `MemeFinder.app`（含 `Info.plist`、網路與檔案存取所需設定）。
- 不需安裝完整 Xcode；環境已具備 Swift 6.3.2 / Apple Silicon / macOS 26.5。

## 10. 開放項目 / 後續階段

- V2：接線上來源（Tenor 優先，與 Gemini 同生態）。
- V2：全域快捷鍵、選單列常駐、以圖搜圖。
